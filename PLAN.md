## 1. 定位:测什么,给谁看

**一句话**:用"人人 10 秒看得懂"的造物任务,评测 **模型 × 编码 harness 组合**(Claude Code、Codex、Cursor Composer、
K2 Code 这类 agent 全家桶)的实战能力,榜单本身做成一个好玩的街机厅网站。

三条边界:
- **测 agent+harness,不测裸模型问答**。弱智吧、数 strawberry 这类题现代 LLM 已经做得很好,没有区分度,砍掉。
  我们的评测对象是"给一句话,agent 自己规划、写码、跑起来、迭代"的全链路能力。
- **通俗性是准入门槛**。任务必须是普通人一眼看懂比什么、成品能在网页里直接看/玩的东西。
- **可玩性即证据**。别的榜单给分数,我们让访客亲手玩 9 个 agent 写的马里奥,好坏自己上手感受,然后盲投。

组合起来。核心差异一句话:**别人展示截图,我们让你玩到成品**。

**赛道准入三关**(每个新点子都要过):
1. 普通人 10 秒内看懂比什么;
2. 成品能在网页上直接展示或试玩;
3. 有区分度(不能所有 agent 都满分或都零分)。

### 新赛道统一协议(马里奥当年没存档,从 Track 2 起严格执行)

  运行的 harness 及版本、日期。
- agent 全自动跑,人不碰键盘;产物目录只读,bug 也不修,页面如实展示。

## 4. 评分与榜单机制

- **作品型赛道**(马里奥、造城):双盲对战投票,访客随机看两件匿名作品(可试玩)选更好的,Bradley-Terry/Elo 聚合
  (WebDev Arena 同款,方法成熟直接抄);配一张客观检查表(能否运行、机制完整度、有无音效)作副榜。
- **自动判定赛道**(弹球):脚本检查硬指标(穿墙、飞出、帧率崩溃),百分制;观赏性另设投票副榜。
- **主榜「街机总分」**:各赛道 rank 百分位等权平均;赛道少时弱化主榜,以赛道页为主。
- **防刷**:指纹去重 + 频控 + 单 IP 日上限;初期流量小先做最简版。

## 5. 前端:AA 风格,内容本身负责好玩

**主题:Artificial Analysis 式白底图表风**(用户 2026-07-10 拍板,推翻此前的像素街机厅方案)。
视觉专业干净、图表优先,"好玩"由可玩内容承担,不靠皮肤:

- **首页**:顶栏 + 一句话定位 + 指标块(参赛组合/赛道数)+ 赛道卡片一览(未上线标 coming soon)+
  马里奥榜单图表 + 方法论(prompt 原文、协议披露)。
- **马里奥赛道页**:参赛卡片网格,每卡一个「▶ 试玩」按钮弹全屏 iframe 直接玩;客观检查表全员对比表格;
  对战盲投模式(M2 随投票后端一起上)。
- **鹈鹕页**:每个 agent 一条"进化胶片"(v1→vN 缩略图横排);**弹球页**:九宫格同屏观赏 + 穿墙"社死时刻"截帧。
- **分享卡**:每赛道一键生成战报图(AA 风卡片,自带二维码)。
- 每个成品放 sandbox iframe,互不污染;图表一律内联 SVG,零外部依赖。

**技术栈**:Next.js 静态导出 + Cloudflare Pages(复用 DRA 部署经验,本工作站推送);
投票后端 Cloudflare Workers + D1(M2 再做,MVP 先不开投票)。

## 8. 风险与规矩

1. **任天堂版权**:对外统一叫"一句话平台跳跃游戏",不用任天堂商标图;逐个成品过素材来源(fable5 自称全原创,其余 8 个未查)。
2. **协议不齐**:马里奥赛道生成协议没存档,页面必须如实披露;从 Track 2 起协议先行。
3. **模型命名**:目录名是内部代号(5.6sol 等),对外需映射正式"模型 × harness"名,`meta.json` 里补。
4. **别学术化**:KPI 是"好玩、想转发"。rigor 只保底线(只读、prompt 存档、双盲、协议公开),其余为趣味让路。
5. **跑新赛道要花钱**:每赛道 = 9 个 agent 各跑一次,按 harness 计费;弹球最便宜先跑,造城最贵最后跑。
