Methodology

方法论

rigor 只保底线:prompt 存档、目录只读、协议公开、双盲投票。其余为趣味让路。

「帮我生成一个马里奥小游戏」 原始 prompt 逐字原文,单 prompt 简单粗暴,已由用户确认存档(2026-07-10)

协议要点

  • 所有参赛者收到逐字相同的同一句 prompt;
  • agent 全自动规划、写码、迭代,人不碰键盘;
  • 成品零人工修改,发现 bug 也不修,页面如实展示;
  • 成品目录只读,评测可信度全靠这一条铁律。

如实披露

  • 马里奥赛道跑于协议规范建立之前:各 harness 的具体版本与迭代轮数当时未存档,无法追溯;
  • 自下一条赛道(弹球万花筒)起,协议先行:prompt、迭代规则、harness 版本、日期全部先存档再开跑;
  • 9 个参赛名称为内部代号,正式「模型 × harness」对照表待补。
